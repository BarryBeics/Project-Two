Detailed Explanation

There is a phenomenon called "momentum bias" a human flaw you could say that when things are going well we believe they will keep getting better and when things are bad we feel they are just going to keep gettting worse.

This can be observed in financial markets as described here: Referenced from wikipedia "In finance, momentum is the empirically observed tendency for rising asset prices to rise further, and falling prices to keep falling"

Therefore the tool allows the user to explore the relationship between the phenomenon and adjusting certain trading parameter to seek out opoptunties for profits in trading.

This is acheved with the following steps.

1, Market simulation --

The code running this tool uses a number of for-loops to move a price from any starting value to a end value which is set by choosing a "time period" and a "percentage" of change. e.g. if a currency had a start value of 1000 and a move to the up of 6% over a 1 hour period this would give us a end price of 1060 Therefore we could say the average move per minute is 1 The script then increments the price by 1 each minute to give us an average moving price. In order to provide a more realistic simulation of how prices can fluxuate the script has the ability to increase of decrease the price randomly off this average move price.

To maintain control of the market momentum for the purpose of this tool but still offer a realistic market conditions we then combine this average move with the randon fluxuations, this ensure we reach the end value as desired but still provide market volatility.

2, Volatility --

The user is offered the option to select the desired percentage of volatility so that a) They can define the conditions they would enter a trade b) They can see how this chosen volatility would work in relations to the chosen Stop Loss & Take Profit amounts.
